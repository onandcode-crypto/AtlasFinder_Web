5-4. 예약 현황 조회

항목
내용
조회 방법

조회 결과 표시
촬영 유형, 희망 날짜, 현재 상태 (대기중 / 확정 / 취소)
복수 예약 처리

조회 보안
입력값이 DB에 일치하는 경우에만 결과 노출 (존재 여부 노출 최소화)
💡 전화번호/이메일 조회는 완전 일치 방식으로 처리합니다. 개인정보 보호를 위해 일치하지 않을 경우 '조회 결과가 없습니다'로만 안내합니다.
5-5. 예약 가능 달력
🆕 달력에서 예약 가능한 날짜만 선택 가능하도록 표시
항목
내용
기본 예약 가능 기간
오늘 기준 최대 1개월 뒤까지 예약 가능
관리자 설정
관리자 대시보드에서 공개 기간을 직접 설정 가능 (예: 2주 / 1개월 / 2개월 등)
불가 날짜 처리
관리자가 특정 날짜를 예약 불가로 지정 가능 (휴무, 마감 등)
달력 표시
예약 가능일: 활성화 / 예약 불가일: 회색 처리 / 이미 지난 날: 비활성화
표시 단위
월 단위 달력 UI, 날짜 클릭으로 선택
💡 관리자가 공개 기간을 줄이면 기존 예약은 유지되고, 새 예약만 단축된 기간 내에서만 가능합니다.
5-6. 예약 알림 플로우
단계
내용
① 신청자 제출
예약 신청 폼 제출 (이름, 전화번호, 이메일, 촬영유형, 날짜 등)
② 자동 접수 이메일
신청자 이메일로 접수 완료 안내 자동 발송
③ 관리자 알림 이메일
관리자 이메일로 새 예약 신청 내용 알림
④ 관리자 상태 변경
대시보드에서 대기 → 확정 / 취소로 변경
⑤ 확정/취소 안내 이메일
상태 변경 시 신청자에게 결과 안내 이메일 자동 발송
💡 Gmail SMTP + Nodemailer 조합. Gmail 하루 500건 발송 제한 — 예약 건수 증가 시 SendGrid 전환 고려.
